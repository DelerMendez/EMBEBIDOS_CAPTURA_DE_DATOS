\doxysubsection{Detailed Description}
Módulo central de inicialización y arranque de la aplicación. 

Este archivo se encarga de:\+
\begin{DoxyItemize}
\item Configurar la gestión de energía del sistema (Power Management)
\item Inicializar los módulos principales (radio, display)
\item Crear recursos de Free\+RTOS (colas)
\item Lanzar las tareas principales del sistema (sensor y display)
\end{DoxyItemize}

La arquitectura sigue un modelo basado en tareas concurrentes donde:\+
\begin{DoxyItemize}
\item Una tarea adquiere datos del sensor (IMU)
\item Otra tarea se encarga de la visualización
\item Ambas se comunican mediante una cola de Free\+RTOS
\end{DoxyItemize}

\begin{DoxyNote}{Note}
Este módulo debe ser llamado una sola vez al inicio del sistema. 

Las funciones aquí son bloqueantes en caso de error crítico (loop infinito).
\end{DoxyNote}

Punto de entrada principal para iniciar la aplicación. 

Inicia la ejecución de la aplicación.

Esta función realiza:\+
\begin{DoxyItemize}
\item Inicialización de power management
\item Inicialización de comunicación (ESP-\/\+NOW)
\item Inicialización del display
\item Creación de la cola de datos del IMU
\item Creación de tareas Free\+RTOS (sensor y display)
\end{DoxyItemize}

En caso de error crítico, el sistema entra en un bucle infinito mostrando el error en pantalla.

\begin{DoxyNote}{Note}
Función bloqueante durante la inicialización. 

No es thread-\/safe y debe llamarse desde el contexto principal (app\+\_\+main). 
\end{DoxyNote}
